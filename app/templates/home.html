Acerca de la Aplicación

Esta aplicación ha sido reorganizada siguiendo el patrón MVC (Modelo-Vista-Controlador) para mejorar su mantenibilidad y escalabilidad.

Estructura del Proyecto

  • Modelos (app/models/): Representan los datos y la lógica de negocio.
  • Vistas (app/templates/): Interfaces de usuario para la interacción.
  • Controladores (app/routes/): Gestionan las peticiones y conectan modelos y vistas.
  • Servicios (app/services/): Proveen funcionalidades reutilizables entre componentes.

Este enfoque permite una clara separación de responsabilidades y hace que el código sea más fácil de mantener y extender.
